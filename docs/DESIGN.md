# 设计文档：从伤寒-赫尔墨斯到诗海赫尔墨斯

本项目把 Shanghan-Hermes（《伤寒论》自主规则挖掘系统）的架构完整移植到
古典诗词域。移植的原则：**保留全部工程不变量（证据闸门、引用核验、
确定性离线后端、能力代理、假健康防护），替换全部领域件（证据单位、
词库、规则类型、专家分工）**。

## 二、原样保留的工程不变量

1. **证据闸门**（`review/gates.py`）：schema → 逐字回源（`contains_verbatim`，
   简繁异体折叠后判定）→ 语义 → 对抗批评 → 一轮只删不增的修复 + 重核 →
   共识评分（证据占 0.50 权重）→ 发布分级（gold/silver/bronze/rejected）。
   无证据或批评硬失败 → 一律拒绝。三个落盘口：rules_initial / rejected / audit。
2. **CitationGuard**（`agent/citation.py`）：存在性 + 逐字引文 + 就近归属 +
   取证闭环（存在≠取证，空证据 fail-closed）。守卫是控制器不是装饰器：
   违规裁定回灌给智能体重写，最多两轮。
3. **确定性 local 后端**（`llm/providers.py`）：与真实模型相同的两步工具循环
   （路由→组稿），组稿只引用工具结果中的 poem_id；全系统离线可跑、可测、
   逐字节可复现；真实模型失败优雅回退。
4. **能力代理 ToolRegistry**（`agent/tools.py`）：默认拒绝未知工具、参数矫治、
   schema 校验、结果大小护栏、环形审计；ScopedRegistry 按角色裁剪工具面
   （能力层最小权限，非提示词约束）。
5. **假健康防护**（`health.py`）：livez/readyz 分离；readyz 校验 poems 行数
   与 manifest 逐一相符；assert_ready 在注册表构建前响亮拦截。
6. **检索工程**（`rag/`）：字 unigram+bigram BM25 → 5× 候选池归一到 0-10 分带
   → 实体（意象/题材）覆盖度重排 → 直接引用捷径（《题名》/poem_id 得分 99）。
7. **文本归一**（`textutil.py`）：OpenCC 单字表构建保长度 translate 映射，
   折叠后匹配偏移仍有效；bigram Dice 相似度做引文模糊匹配与对齐。

## 三、领域特有的新设计

- **简繁双轨**：语料本身简繁混杂（全唐诗繁体、宋词简体）。原文永远原样
  保存（A 层不改字），索引与匹配一律在 t2s 折叠层进行。
- **跨集去重（重出互见）**：以「简体作者+正文前 12 字」为键；精选集先登记
  故成为保留方，后见记录并入 `also_in` 并合并 tags/notes/导读。系统内
  剩余的近重复（版本异文，如「床前明月光/床前看月光」）不强行去重——
  它们本就是文献学意义上的异文，由互文检测如实标注为「重出互见」。
- **韵伴聚类的诚实口径**：探测不到可靠的平水韵开放数据时，不手工编码
  半可靠韵表冒充权威，而是从语料自身归纳。口径经对抗审核迭代为：
  只取 4/8 句、5/7 言近体近似样本；**相邻韵脚连边**（换韵古体混入时
  只产生一条权 1 弱边被剪除，真律诗经传递闭包成组）；边权≥2 剪枝；
  超大分量如实标注 low_purity。实测 53 组平/入声零混杂（「竹·宿」
  入声一屋组、「年·天·烟」一先组）。每处输出注明「语料归纳，
  非平水韵权威表」。
- **D 层双向回源**：外部 LLM 分析样本必须同时满足（a）作者一致、
  （b）分析文本逐字包含该诗首句，才允许绑定；绑定后仍标注
  「外部 LLM 生成，非本系统结论」。样本通过 HF datasets-server filter API
  按本语料 536 位作者定向抓取（4,967 条 → 绑定 3,084 首）。
- **C 层训诂（gujilab）**：说文解字逐字条目 + 尔雅训释组构成字义层，
  `poetry_gloss` 工具与作品全息抽屉可查；训诂标注「字书本义，诗中用义
  可能引申」——不越层作诗义定论。
- **SkillRAG 自我消费**：编译出的 Skill 树（SKILL.md + examples.jsonl）
  被系统自身装载为问题路由器：示例命中优先、BM25 描述兜底；路由提示
  注入智能体 system 消息，local 与真实大模型两个后端同源受益。

## 四、评测设计（自监督+金标）

- retrieval：名句→原诗（Top-1 96.0%）；孪生命中（重出互见）单独计数。
- metrics：体裁计量 vs 语料自带标签（97.7%）——真正带金标的评测；
  不一致项（古绝/折腰体）正是「计量近似」边界的如实呈现。
- grounding：自动问题库跑智能体，引用核验通过率 95.8%（local 后端）。
